# 실행 전 확인 규칙

이 파일은 이 저장소에서 작업할 때 실행 전에 확인하는 기본 규칙입니다.

## 필수 원칙

1. 모든 파일은 UTF-8 인코딩으로 작성/수정한다.
2. 문서화는 항상 수행한다.
3. DB 기본 구성은 Neon PostgreSQL + Prisma를 기준으로 한다.
4. 설정 변경 시 관련 문서를 함께 업데이트한다.
5. CSS 수정(특히 margin/padding/width/overflow) 시 기존 스타일과 충돌 여부를 항상 점검하고 검증한다.
6. front 토스 디자인 사용
7. 아이템/섹션 컴포넌트 내부 패딩은 공통 변수로 관리하고, 상하좌우 값을 개별 하드코딩하지 않는다.
8. 공통 UI(nav, 탭, 카드 등)의 font-size, padding 수정 시 전역 CSS 변수/컴포넌트 클래스 충돌 여부를 반드시 확인하고 검증한다.
