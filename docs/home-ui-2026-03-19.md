# 홈 화면 뉴스 선정 기준

## 문서 목적

이 문서는 홈 화면에서 `맞춤 뉴스`, `헤드라인`, `실검 뉴스`가 어떤 기준으로 선정되고 노출되는지 정리한다.

## 맞춤 뉴스

### 노출 조건

- 홈 탭이 `맞춤 뉴스`일 때만 노출한다.

### 후보 데이터

- `useInfiniteNews()`로 현재까지 불러온 전체 뉴스 목록을 후보군으로 사용한다.

### 선정 기준

- `rankPersonalizedNews(articles, profile)` 결과를 그대로 사용한다.
- 개인화 프로필은 익명 사용자 기준이며, 카테고리 선호도, 키워드 반응, 최근 행동 이력이 반영된다.
- 같은 후보군 안에서는 개인화 점수가 높은 뉴스가 먼저 노출된다.

### 요약 노출 기준

- 각 뉴스 카드에는 3줄 요약을 노출한다.
- `/api/ai/summarize` 응답이 있으면 AI 요약을 사용한다.
- 요약이 아직 없으면 기본 안내 문구 3줄을 임시로 노출한다.

### 페이징 기준

- 최초 8건을 노출한다.
- `맞춤 뉴스 더 보기` 클릭 시 8건씩 추가 노출한다.
- 현재 로컬 후보군을 모두 소진했고 `useInfiniteNews()`에 다음 페이지가 있으면, 다음 뉴스 페이지를 먼저 불러온 뒤 추가 노출한다.

## 헤드라인

### 노출 조건

- 홈 탭이 `헤드라인`일 때만 노출한다.

### 후보 데이터

- `useInfiniteNews()`로 현재까지 불러온 전체 뉴스 목록을 사용한다.

### 선정 기준

- 현재 로드된 뉴스 배열의 앞쪽 5건을 그대로 사용한다.
- 별도의 개인화 점수나 추가 가중치는 적용하지 않는다.
- 정렬은 서버에서 내려온 뉴스 순서를 유지한다.

### 노출 개수

- 항상 최대 5건만 노출한다.

## 실검 뉴스

### 노출 조건

- 홈 탭이 `실검`일 때만 노출한다.

### 후보 데이터

- `useInfiniteNews()`로 현재까지 불러온 전체 뉴스 목록을 사용한다.
- 기사 제목과 설명에서 추출한 키워드 기반 후보를 함께 사용한다.

### 키워드 추출 기준

- 제목과 설명 문자열을 합쳐 소문자 기준으로 정규화한다.
- 특수문자를 제거하고 공백 단위로 분리한다.
- 길이가 2자 미만인 토큰은 제외한다.
- 불용어는 제외한다.
  - 예: `오늘`, `이번`, `관련`, `기자`, `뉴스`, `정부`, `시장`, `오전`, `대한민국`, `속보`, `단독`

### 선정 기준

- 먼저 현재 뉴스 목록 상위 10건에 기본 헤드라인 점수를 부여한다.
  - 앞에 있을수록 더 높은 점수를 준다.
- 같은 뉴스가 반복 키워드에 걸리면 키워드 출현 빈도와 키워드 순위에 따라 추가 점수를 준다.
- 최종 점수 기준으로 내림차순 정렬한다.
- 점수가 같으면 `publishedAt`이 더 최신인 뉴스를 우선한다.

### 노출 개수

- 최종 상위 10건을 노출한다.
