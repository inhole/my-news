# 홈 UI 개편 기록 (2026-03-24)

## 목적

`frontend-skill` 기준을 홈 화면에 적용해 첫 화면 인상을 강화하고, 카드 나열 중심 구조를 에디토리얼형 레이아웃으로 재편했다.

## 적용 내용

### 1. 홈 첫 화면을 리드 스토리 중심으로 재구성

- 기본 홈 탭(`tab=weather`)을 사실상 `오늘` 개요 화면으로 사용한다.
- 첫 화면에 대표 기사 1건을 전면 히어로로 배치한다.
- 히어로 안에서 브랜드, 기사 카테고리, 메타 정보, CTA를 한 구도로 정리한다.
- 상세 날씨 위젯은 홈 본문에서 제거하고, 상단 네비게이션에서 날짜 옆 미니 요약 정보로 축소했다.
- 상단 날씨 요약을 클릭하면 `/weather` 상세 페이지로 이동한다.

### 2. 요약 섹션 구조 정리

- 히어로 아래에 `Headlines`, `Trending`, `For You` 브리프 영역을 둔다.
- 각 섹션은 하나의 역할만 갖도록 구성한다.
- 주요 뉴스 리스트는 개별 카드 분리 대신 기존 뉴스 리스트처럼 분리선 중심 리스트로 유지한다.
- 대신 서로 다른 주제의 상위 섹션 카드 사이 간격을 더 분명하게 둔다.

### 3. 기존 기능 로직 유지

- `useInfiniteNews()` 기반 데이터 로딩 구조는 유지한다.
- 개인화 정렬, 트렌딩 키워드 계산, 무한 스크롤은 기존 로직을 그대로 사용한다.
- 디자인만 강하게 바꾸고 데이터 흐름과 추천 규칙은 회귀되지 않도록 했다.

### 4. 상단 탐색 라벨 정리

- 홈 탭 라벨을 `오늘`, `맞춤`, `헤드라인`, `트렌딩`으로 정리했다.
- `weather` 탭은 날씨 전용 탭이 아니라 홈 개요 탭 역할을 맡는다.

## CSS 원칙

- 패딩은 기존 공통 변수(`--section-pad-*`, `--item-pad-*`)를 그대로 활용한다.
- 새 홈 UI용 클래스는 전역 변수와 충돌하지 않도록 `home-*`, `editorial-*` 접두사로 추가했다.
- 히어로, 브리프, 섹션 reveal 애니메이션은 과하지 않은 한 방향 모션만 사용한다.
