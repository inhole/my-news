# 맞춤 뉴스 요약 출력 수정

## 변경 일시

- 2026-03-24

## 문제

- 홈의 `맞춤 뉴스 리스트`에서 기사에 저장된 요약이 즉시 읽기 좋은 형태로 노출되지 않았다.
- 기존 구조는 `summaryLines` 배열 중심이라 리스트에서 짧은 요약문으로 다루기 어려웠다.

## 원인

- `my-news-front/lib/personalization/personalized-feed.ts`의 `rankPersonalizedNews()`가 정렬 결과를 만들 때 원본 기사 객체의 요약 데이터를 덮어쓰고 있었다.
- 리스트 렌더링이 `summaryLines` 배열 표시 방식에 묶여 있어, 요구사항인 "120자 이내 단일 요약문" 구조와 맞지 않았다.

## 조치

- 개인화 정렬 결과에서 원본 요약 데이터를 유지하도록 수정했다.
- 뉴스 배치 수집 시점에만 OpenAI 요약을 생성하고 DB에 저장하도록 정리했다.
- 백엔드 응답에 `summary` 필드를 추가하고, 저장된 `summaryLines` 또는 설명/본문을 기반으로 120자 이내 단일 요약문을 내려주도록 수정했다.
- 맞춤 뉴스 리스트는 배열 대신 `summary` 단일 텍스트를 우선 노출하도록 변경했다.

## 영향 범위

- 홈 `맞춤` 탭 리스트
- 홈 개요의 `For You` 섹션
- 뉴스 목록/상세 API 응답의 요약 형식

## 확인 포인트

- 맞춤 뉴스 리스트는 `summary` 필드를 우선 사용해 120자 이내 요약문을 표시해야 한다.
- 맞춤 뉴스 리스트의 요약문은 `line-clamp` 없이 카드 내에서 전체 텍스트가 보이도록 표시해야 한다.
- 기존에 요약이 비어 있는 기사는 뉴스 배치 실행 후 DB에 요약 데이터가 저장되어야 한다.
- `summary`가 비어 있을 때만 설명문을 제한적으로 대체 표시해야 한다.
