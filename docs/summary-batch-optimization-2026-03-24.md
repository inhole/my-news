# 요약 배치 호출 최적화

## 변경 일시

- 2026-03-24

## 목적

- OpenAI 호출 횟수를 줄이고 한 번의 요청으로 더 많은 뉴스를 요약한다.

## 조치

- 요약 배치 크기를 고정 5건에서 동적 배치로 변경했다.
- 한 배치에는 최대 20건까지 담되, 프롬프트 전체 길이가 과도해지면 다음 배치로 넘긴다.
- 기사별 입력 길이를 제한했다.
  - 제목: 120자
  - 설명: 240자
  - 본문: 800자

## 기대 효과

- 동일한 뉴스 수 기준 OpenAI 호출 횟수 감소
- 프롬프트 길이 초과 위험 완화
- 요약 품질에 필요한 핵심 정보는 유지하면서 비용과 지연 감소
