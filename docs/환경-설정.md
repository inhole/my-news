# 환경 설정

이 프로젝트는 서버 환경을 `로컬`, `운영` 두 가지로만 관리합니다.

## 로컬 환경

### Frontend

1. `my-news-front/.env.local.example`을 참고해 `my-news-front/.env.local`을 작성합니다.
2. 기본값:

```env
NEXT_PUBLIC_API_BASE_URL=/api
BACKEND_BASE_URL=http://localhost:3000
```

3. 실행:

```bash
npm run dev:front
```

### Backend

1. `my-news-back/.env.local.example`을 참고해 `my-news-back/.env`를 작성합니다.
2. 필수값을 채웁니다.

```env
NODE_ENV=local
PORT=3000
CORS_ORIGIN=http://localhost:3001
ENABLE_SWAGGER=true
ENABLE_NEWS_SCHEDULE=true
ENABLE_STARTUP_NEWS_FETCH=true
DATABASE_URL=postgresql://...
NAVER_CLIENT_ID=...
NAVER_CLIENT_SECRET=...
```

3. 실행:

```bash
npm run dev:back
```

## 운영 환경

### Frontend

1. `my-news-front/.env.production.example`을 기준으로 `my-news-front/.env.production`을 작성합니다.
2. `BACKEND_BASE_URL`은 실제 API 도메인으로 설정합니다.

### Backend

1. `my-news-back/.env.production.example`을 기준으로 `my-news-back/.env.production`을 작성합니다.
2. 권장값:

```env
NODE_ENV=production
ENABLE_SWAGGER=false
ENABLE_STARTUP_NEWS_FETCH=false
ENABLE_NEWS_SCHEDULE=true
CORS_ORIGIN=https://your-front-domain.com
```

## 실행 명령

```bash
npm run build
npm run start:prod
```
